# Docs Writing Rule 文档编写规则

Docs Writing Rule 文档编写规则用于定义当前 `apps/web-docs/content/docs` 下所有产品规划文档应共同遵循的表达规则。

## Core Rules 核心规则

- 第一次出现的英文术语必须附带中文解释
- 关系名称必须使用 `English + 中文语义` 形式
- 字段名称必须附带中文定义
- 如果章节标题是技术术语，章节第一句必须用中文解释它的含义
- 不要在 foundation 基础层 或 product 产品层 文档中留下未解释的纯英文术语
